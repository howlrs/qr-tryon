# 実装計画


## 開発手順
1. NextjsでAPI含むフロントエンドを構築する
2. 実働を確認する
3. APIを切り分ける
4. バックエンドを構築する
5. バックエンドを確認する
6. 統合し、動作を確認する

## フロントエンド
- Nextjsを使用する
- フロントエンドを構築する
- フロントエンドを確認する
- 要件を確認するため、Nextjs上にAPIを実装しモック状態でAPI連動稼働状態を実現する（※開発手順2まで）
- 開発手順3以降、Nextjs上のAPIを要件としてGoでAPIを構築する
- 開発手順5以降、バックエンドを確認する

## API
- 速度と型言語として、Goを使用する
- Models, Services, Repositories, Handlersを分けて構築する
- データベースとの接続はBunを使用する
- バックエンドを確認する


## データベース
- PostgreSQLをメインデータベースとして採用
- データの永続化とトランザクション管理を担う
- スキーマ設計は正規化を基本とし、パフォーマンス向上のためのインデックスを適切に設定する
- Goからの接続には、ORM（例: GORM）または`database/sql`パッケージを使用する
- 開発環境と本番環境で異なる設定（接続情報、レプリケーションなど）を考慮する
